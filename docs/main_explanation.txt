Annotated explanation of src/main.cpp

This document explains the top-level structure, functions, variables and the main rendering loop in `src/main.cpp`.

Includes and headers
--------------------
- The file begins by including standard C headers and C++ headers used for IO, math, and containers.
  - `<cstdio>`, `<cstdlib>`, `<cmath>`: C standard libs for formatted IO, general utilities, math
  - `<cfloat>`: floating point limits (uses `FLT_MAX`)
  - `<iostream>`, `<fstream>`: C++ streams for console and file IO
  - `<algorithm>`, `<vector>`: container and algorithm utilities

- Project headers included:
  - `camera.hpp`: declares `Camera` and `get_ray()` used to generate rays for each pixel
  - `world.hpp`: scene container type `World` with `hit()` to find intersections
  - `sphere.hpp`: `Sphere` geometry containing center, radius, and color
  - `vec3.hpp`: `Vec3` 3D vector class used for points, directions, and colors

Helper functions
----------------
1) `void write_color(std::ostream &out, const Vec3 &color)`
   - Converts a `Vec3` color (components in [0,1]) to three 0..255 integers and writes them as text `R G B` to the provided stream.
   - Uses `std::max`/`std::min` to clamp the color channels to [0,1], multiplies by `255.999f` and casts to `int`.
   - Not used for binary PPM output in the current code, but common for ASCII PPM (P3).

2) `Vec3 unpackRGB(unsigned int packedRGB)`
   - Input: 24-bit color packed as `0xRRGGBB` in an `unsigned int`.
   - Extracts red, green, blue components by bit-shifting and masking:
       - `r = (packedRGB >> 16) & 0xFF`
       - `g = (packedRGB >> 8) & 0xFF`
       - `b = packedRGB & 0xFF`
   - Returns `Vec3{r/255.0f, g/255.0f, b/255.0f}` converting each channel to normalized float in [0,1].

3) `bool in_shadow(const Ray &shadow_ray, const World &world, float max_t)`
   - Tests whether a `shadow_ray` is occluded by any object in `world` before distance `max_t`.
   - Calls `world.hit(shadow_ray, 0.001f, max_t, temp)` where `0.001f` is used to avoid self-intersection (shadow acne) and `temp` is a temporary `HitRecord`.

4) `Vec3 shaded_color(const HitRecord &rec, const Vec3 &light_pos, float light_brightness, const World &world)`
   - Computes diffuse shading for the hit record `rec`.
   - Steps:
     - `to_light = normalize(light_pos - rec.point)` creates a unit vector pointing toward the light.
     - `dist2 = distance2(light_pos, rec.point)` squared distance from hit point to light.
     - `intensity = light_brightness * max(0, dot(to_light, rec.normal)) / dist2` applies inverse-square falloff and Lambertian cosine.
     - `intensity = min(1.0f, intensity)` clamps intensity to 1.0.
     - A `shadow_ray` starting at `rec.point + rec.normal * 0.001f` along `to_light` is cast; if blocked before the light distance, `intensity` is multiplied by 0.1 (in-shadow dimming).
     - Returns `rec.color * intensity` where `rec.color` is the surface base color (Vec3).

   - Notes:
     - `distance2` is squared distance; used with inverse-square falloff.
     - The code uses a small epsilon `0.001f` to avoid the ray hitting the same surface it started from.

5) `Vec3 ray_color(const Ray &r, const World &world, const Vec3 &background_color, const Vec3 &light_pos, float light_brightness)`
   - Traces a single ray `r` through the scene.
   - If it hits an object (`world.hit(r, 0.001f, FLT_MAX, rec)`), returns `shaded_color(rec, ...)` computed from the hit record.
   - If no hit, returns `background_color`.

6) `static unsigned char to_byte(float v)`
   - Clamps the float `v` to [0,1] then converts to an unsigned 8-bit value in 0..255 by multiplying by `255.999f` and casting.
   - Used immediately before writing binary RGB bytes to the P6 PPM file.

Main function walk-through
--------------------------
Signature: `int main(int argc, char *argv[])`
- Expects exactly two command-line arguments: `input.txt` and `output.ppm`.
- Usage printed and returns `1` if argument count is wrong.

Input file opening and checks
- Reads the input scene file with `std::ifstream in(argv[1])` and fails with an error message if the file can't be opened.

Variables declared (top-level):
- `int image_width, image_height` — image resolution in pixels.
- `float viewport_height` — height of the camera viewport in world units.
- `float focal_length` — focal length of the camera (distance from projection plane).
- `Vec3 light_pos` — position of the point light.
- `float light_brightness` — brightness scalar used in shading equation.
- `int color_count` — number of palette colors that follow.
- `std::vector<unsigned int> colors` — stores packed hex color integers (0xRRGGBB).
- `int background_index` — index into palette for background color.
- `int sphere_count` — number of spheres described next in the file.

Reading input fields (order matters)
- `in >> image_width >> image_height;`
- `in >> viewport_height;`
- `in >> focal_length;`
- `in >> light_pos.x >> light_pos.y >> light_pos.z >> light_brightness;`
- `in >> color_count;` then `colors.resize(color_count);`
- For each color read as a string (e.g. `0xRRGGBB`) and parsed with `std::sscanf(hex.c_str(), "0x%06x", &value)` storing the integer in `colors[i]`.
- `in >> background_index;`
- `in >> sphere_count;`

Input validation
- The code checks `background_index` to ensure it is within `[0, color_count-1]` and prints an error if invalid. (If invalid the program returns 1.)

Palette construction
- `std::vector<Vec3> palette(color_count);` allocates palette entries as normalized `Vec3` colors using `unpackRGB(colors[i])`.
- `Vec3 background_color = palette[background_index];`

Camera and world setup
- `Camera camera(image_width, image_height, viewport_height, focal_length);` constructs a camera that will produce rays for each pixel via `camera.get_ray(u, v)`.
- `World world;` is the scene container; objects will be added via `world.add(&spheres.back());` after spheres are created.

Reading spheres
- A `std::vector<Sphere> spheres;` is reserved with capacity `sphere_count`.
- For each sphere, the code reads `center` (Vec3), `radius` (float) and `color_index` (int).
- Validates `color_index` is within range; if invalid prints error and exits.
- `spheres.emplace_back(center, radius, sphere_color);` constructs a `Sphere` with the chosen color; `world.add(&spheres.back());` adds a pointer to the freshly created sphere into the `World` container.

Output file and PPM header
- `std::ofstream out(argv[2], std::ios::binary);` opens the output PPM file in binary mode.
- Writes the binary PPM header `P6\n<width> <height>\n255\n` which indicates a binary PPM with 8-bit color channels.

Anti-aliasing sample pattern
- `const float sample_offsets[3] = {1.0f/6.0f, 3.0f/6.0f, 5.0f/6.0f};` defines 3 sub-pixel offsets per axis, achieving 3x3 = 9 samples per pixel.
- The offsets are at 1/6, 3/6, 5/6 across a pixel which centers samples roughly evenly in the pixel area.

Main render loop (pixel sampling)
- Loops scanlines `for (int j = image_height - 1; j >= 0; --j)` to write rows top-to-bottom as PPM expects top first.
- For each pixel column `i`:
  - Accumulate `Vec3 pixel_color{0,0,0}`.
  - For each sub-sample `sy` and `sx` in `0..2`:
    - Compute normalized coordinates u and v as:
        - `u = (i + sample_offsets[sx]) / (image_width - 1)`
        - `v = (j + sample_offsets[sy]) / (image_height - 1)`
      These map pixel+subsample positions into [0,1] ranges across the viewport.
    - Generate a ray `Ray r = camera.get_ray(u, v);` for that sample.
    - `pixel_color += ray_color(r, world, background_color, light_pos, light_brightness);` accumulate sample color.
  - After the 9 samples, `pixel_color = pixel_color / 9.0f;` averages samples to get final color.
  - Convert each channel to a byte using `to_byte()` and write three bytes to `out` with `out.write(reinterpret_cast<char*>(pixel), 3);`.

Finalization
- Close the output file with `out.close();`.
- Print `Rendered <output>` message to stdout and return 0.

Key types and symbols used from other headers
-------------------------------------------
- `Vec3` (from `vec3.hpp`): Represents 3-component vectors and colors. Common operations used here: `+`, `-`, `*` scalar, `/` scalar, `dot()`, `normalize()`, `distance2()`.
- `Ray` (from `ray.hpp`): Ray has an origin and direction; camera produces `Ray` objects; `Ray::at(t)` returns point along ray.
- `Camera` (from `camera.hpp`): Constructed with image resolution, viewport size, focal length; `get_ray(u, v)` returns a `Ray` going through the viewport coordinate `(u,v)`.
- `World` (from `world.hpp`): Stores `Hittable*` objects and provides `hit(ray, t_min, t_max, rec)` that populates a `HitRecord` with intersection data.
- `HitRecord` (from `hittable.hpp`): Contains `point`, `normal`, `color`, `t` of the hit.
- `Sphere` (from `sphere.hpp` / `sphere.cpp`): Implements `Hittable::hit(...)` returning `rec.color` and other intersection data.

Implementation notes and possible improvements
-------------------------------------------
- Color sorting: earlier code used `qsort` to order palette entries — current code relies on color order as provided in the scene file. If you want deterministic sorted palettes, reintroduce a stable sort on `colors` before `unpackRGB`.
- Gamma correction: current renderer writes linear RGB averaged values directly; for display correctness you may want to apply gamma (e.g., gamma=2.2) by raising each channel to `1/gamma` before converting to byte.
- Anti-aliasing: 3x3 samples is fixed; you can parameterize sample pattern or replace with random jittered sampling for stochastic anti-aliasing.
- Performance: per-pixel loop is not parallelized — consider multithreading (OpenMP or std::thread) for faster renders.
- Error handling: more robust parsing and validation would help (e.g., check `in.good()` after reading blocks).
